# Dev Context

Active when building features and writing code.

## Behavior

- Code first, iterate fast
- Make changes, run tests
- Fix issues as they arise
- Commit frequently

## Priorities

1. Working code > perfect code
2. Tests alongside implementation
3. Small, focused commits
4. Keep momentum

## Allowed Actions

- Create and modify files
- Run commands
- Install dependencies
- Make commits

## Quality Gates

Before commit:
- [ ] TypeScript passes
- [ ] Lint passes
- [ ] Tests pass
- [ ] No console.log left
- [ ] No TODO placeholders

## When Stuck

1. Check error message fully
2. Search codebase for similar patterns
3. Try simpler approach
4. Ask for clarification if needed

## Session Flow

```
Start → Pick task → Implement → Test → Commit → Next task
                        ↓
                    If fails → Fix → Test again
```
